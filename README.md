# 🍃 Komorebi (木漏れ日)

**Komorebi** 是一款基于 **FFmpeg** 的智能化视频压制与转换脚本。

其名字取自日语“木漏れ日”，意为“从树叶缝隙中透过的阳光”。正如其名，本脚本旨在通过精细的参数筛选，滤除冗余的视频数据，在**画质（Fidelity）**、**体积（Size）**与**速度（Speed）**之间找到那个完美的平衡点 $P$。

---

## 🌟 核心特性

1.  **动漫级压制 (Anime-Specific)**：预设针对二次元动画优化的编码参数，保护线条边缘与平滑色块，有效防止色带（Banding）产生。
2.  **全能转换 (Universal)**：支持 MP4, MKV, AVI, MOV, GIF, 3GP 等几乎所有 FFmpeg 支持的格式。
3.  **三梯队容灾引擎**：
    * **第一梯队**：尝试 10-bit 硬件加速（NVIDIA/Intel/AMD），追求极致效率与位深。
    * **第二梯队**：若失败，自动回退至 8-bit 硬件加速，确保兼容性。
    * **第三梯队**：若无显卡环境，自动启动 CPU 多核并行压制，确保任务绝不中断。
4.  **路径零错律**：深度优化了对中文路径、空格、括号及特殊字符的识别，支持直接“拖拽”文件。
5.  **环境自检**：脚本启动即扫描系统环境，缺失组件会提供精准的中文报错。

## 🛠️ 环境配置全攻略 (新手必读)

为了让脚本正常工作，你需要配置 **Python** 和 **FFmpeg**。

### 1. 安装 Python 3 (脚本运行环境)

* **第一步**：前往 [Python 官网下载页](https://www.python.org/downloads/)。
* **第二步**：下载 Windows Installer (64-bit)。
* **第三步（最重要！）**：运行安装包时，**务必勾选底部的 `Add Python to PATH`**。
* **第四步**：点击 `Install Now`。
* **验证**：打开命令行（Win+R 输入 `cmd`），输入 `python --version`，看到 `Python 3.x.x` 即成功。

### 2. 安装 FFmpeg (核心压制引擎)

FFmpeg 是处理视频的灵魂。请按照以下步骤配置，**不能直接双击运行**：

1.  **下载**：访问 [gyan.dev](https://www.gyan.dev/ffmpeg/builds/)，找到 **release builds** 部分，下载 `ffmpeg-release-full.7z`。
2.  **解压**：将下载的压缩包解压。建议将文件夹重命名为 `ffmpeg`，并移动到 `C盘` 根目录下，路径应为 `C:\ffmpeg`。
3.  **配置环境变量（让系统认识它）**：
    * 在任务栏搜索框输入“**环境变量**”，选择“编辑系统环境变量”。
    * 在弹出的窗口中点击下方的“**环境变量**”按钮。
    * 在“**系统变量**”列表中找到 `Path` 一行，双击它。
    * 在右侧点击“**新建**”，输入 `C:\ffmpeg\bin`。
    * 一路点击“确定”关闭所有窗口。
4.  **验证**：打开一个新的命令行窗口，输入 `ffmpeg -version`。如果出现一大串文字而非报错，说明你已经成功点亮了引擎。

## 🚀 使用指南

### 1. 获取脚本
将本项目中的 `komorebi.py` 下载到你的电脑任意位置。

### 2. 启动脚本
在脚本所在文件夹的空白处，按住 `Shift` 并右键，选择“在此处打开 PowerShell 窗口”或“在终端中打开”，输入：
```powershell
python komorebi.py
```

### 3. 交互操作
脚本启动后会进入交互模式，你只需要根据中文提示操作：
* **输入模式**：
    * `1`：**动漫压制**（使用高规格参数，压缩率高且画质精美）。
    * `2`：**格式转换**（快速转换，如 MP4 转 GIF 或 AVI 转 MP4）。
* **输入视频**：直接从文件夹里把视频文件**用鼠标拖进**这个黑色的命令行窗口，它会自动填入路径。
* **设置输出**：
    * 如果直接回车，视频会保存在原处，文件名会自动加上 `_Komorebi` 后缀。
    * 你也可以手动输入想要保存的名字。

## 🔬 技术细节

### 编码逻辑逻辑流
1.  **Pre-check**: 检查 FFmpeg 是否在环境变量中。
2.  **GPU Probe**: 运行空载探测命令，识别 `h264_nvenc` (NVIDIA)、`h264_qsv` (Intel) 或 `h264_amf` (AMD)。
3.  **Command Construction**:
    * 动漫模式默认采用 `CRF 20`（视画质需求动态调整码率）。
    * 音频部分采用 `Copy` 策略，无损保留音轨。
4.  **Error Handling**: 使用 Python 的 `subprocess` 模块实时捕获 `stderr`。如果触发显存溢出（Error 0x80000001 等），立即重构命令并调用 `libx264/libx265` 执行软压。

## ❓ 常见问题 FAQ

* **Q: 拖入文件后提示找不到路径？**
    * A: 请确保文件路径中不要包含极其罕见的生僻字或表情符号。一般的中文和空格都是完美支持的。
* **Q: 为什么 GPU 压制比 CPU 快这么多？**
    * A: GPU（显卡）拥有专门处理视频编码的硬件模块，而 CPU 属于通用计算，处理视频时负担较重。
* **Q: 我想压制 4K 视频可以吗？**
    * A: 可以，但如果显存小于 4GB，建议在模式 2 下选择常规转换，或让脚本自动回退到 CPU 模式。
